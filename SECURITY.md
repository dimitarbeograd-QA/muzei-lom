## Известни съображения
- Admin панелът изисква SHA-256 хеширана парола (`tryAdminLogin()`), но
  проверката е изцяло client-side (сравнение спрямо hash, пазен в
  `localStorage`) — всеки посетител с достъп до DevTools може технически
  да заобиколи проверката или да прочете hash-а. Препоръчително е admin
  действия да минават през защитен backend, ако съдържанието стане
  по-чувствително.
- **Известен проблем:** default admin паролата е **"admin"** (виж
  `DEFAULT_HASH` в index.html) — тривиална за отгатване. Смени я веднага
  през панела „Смени парола" след първо влизане.
- **Известен проблем (по-сериозен):** admin панелът позволява запазване на
  реален Anthropic API ключ (`adm-ai-key-input` → `localStorage['mlcms_ai_key']`),
  който после се праща директно от браузъра
  (`anthropic-dangerous-direct-browser-access: true`) — както за admin
  AI генератора, така и за визитьорския AI чат "Алмус". Ключът се пази в
  **plaintext** в localStorage и е четим от всеки с достъп до
  устройството/браузъра (DevTools → Application → Local Storage), както и
  видим в мрежовите заявки. Крадец на ключа може да прави заявки за сметка
  на музея. Препоръка: проксирай Anthropic заявките през собствен backend,
  който пази ключа server-side, вместо да го излагаш в браузъра.
- Ако бъде добавена форма за контакт/коментари, задължително добави
  input validation и anti-spam защита (напр. honeypot поле).
